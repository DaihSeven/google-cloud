## 🧠 1. Conceitos essenciais (explicado simples)

Antes de sair clicando, entende isso — vai evitar 90% dos erros:

### 🌐 VPC (Virtual Private Cloud)

É a **rede privada** dentro do Google Cloud.

* Tudo roda dentro dela (VMs, subnets, etc.)
* É **global**, mas as subnets são regionais

---

### 🧩 Subnet

Divisão da rede (por região):

Exemplo do lab:

* us-east1 → `10.0.0.0/16`
* europe → `10.1.0.0/16`

👉 Cada VM recebe IP daqui

---

### 🔥 Firewall

Controla o tráfego:

* **Ingress** → entrada (ex: ping, SSH)
* **Egress** → saída

⚠️ Regra importante:

> Se não permitir → está bloqueado

---

### 🏷️ Tags (MUITO IMPORTANTE)

Firewall só funciona se a VM tiver a tag correta.

Exemplo:

```
Firewall → tag: rules
VM → precisa ter: rules
```

Se não tiver → **não funciona** ❌

---

### 📡 ICMP

Usado para:

```
ping
```

Se ICMP estiver bloqueado → ping falha

---

# ⚡ 2. Estratégia para terminar rápido

💡 Ordem ideal:

1. Criar VPC
2. Criar subnets
3. Criar firewall (tudo de uma vez)
4. (Se precisar) criar VM com tags

---

# 🛠️ 3. PASSO A PASSO (sem erro)

---

## ✅ PASSO 1 — Criar VPC

Vá em:

```
VPC network > Create VPC Network
```

Preencha:

* **Name**:

  ```
  taw-custom-network
  ```

* Mode:

  ```
  Custom
  ```

---

## ✅ PASSO 2 — Criar subnets

Crie 3:

### 1️⃣

```
Name: subnet-us-east1
Region: us-east1
IP: 10.0.0.0/16
```

### 2️⃣

```
subnet-europe-west1
10.1.0.0/16
```

### 3️⃣

```
subnet-europe-west4
10.2.0.0/16
```

👉 Clique **Create**

---

## 🔥 PASSO 3 — Firewall (FAÇA CERTO AQUI!)

⚠️ ERRO MAIS COMUM: usar rede `default`

👉 Sempre usar:

```
taw-custom-network
```

---

### 🌐 HTTP

```
Name: nw101-allow-http
Tag: http
IP: 0.0.0.0/0
tcp: 80
```

---

### 📡 ICMP (SEU PROBLEMA ANTERIOR)

```
Name: nw101-allow-icmp
Tag: rules
IP: 0.0.0.0/0
Protocol: icmp
```

---

### 🔐 SSH

```
Name: nw101-allow-ssh
Tag: ssh
tcp: 22
```

---

### 🖥️ RDP

```
tcp: 3389
```

---

### 🔄 INTERNAL

```
Name: nw101-allow-internal
Targets: ALL instances
IPs:
10.0.0.0/16
10.1.0.0/16
10.2.0.0/16

Protocols:
tcp: 0-65535
udp: 0-65535
icmp
```

# 🚨 4. Checklist anti-erro (ESSENCIAL)

Antes de clicar "Check my progress":

✔ Criou VPC **taw-custom-network**
✔ Criou 3 subnets
✔ Firewall está na rede correta (não default)
✔ ICMP está como:

```
Other protocols: icmp
```

✔ Tags batem com firewall

---

# 💥 5. Erros clássicos (evite isso!)

### ❌ Regra na rede default

→ não funciona

### ❌ VM sem tag

→ firewall ignorado

### ❌ Esquecer ICMP como "Other protocol"

→ ping não funciona

### ❌ IP range errado

→ comunicação interna falha

---

# 🧠 6. Entendimento rápido (pra prova/entrevista)

Se te perguntarem:

👉 **Por que usar VPC custom?**
→ controle total de IPs e subnets

👉 **Por que usar tags?**
→ aplicar firewall só onde precisa

👉 **Firewall padrão permite ICMP?**
→ só na default network

👉 **Por que ping falha?**
→ ICMP bloqueado ou tag errada

---

# 🚀 Resumo final (modo prova)

* VPC = rede
* Subnet = região + IP range
* Firewall = controle de acesso
* Tag = quem recebe a regra
* ICMP = ping
