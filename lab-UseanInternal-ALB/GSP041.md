# 🌐 Internal Application Load Balancer no Google Cloud

---

## 🧠 1. Conceitos Fundamentais (ANTES do lab)

### 🌐 Load Balancer (Balanceador de Carga)

É um serviço que **distribui requisições entre múltiplas máquinas**, garantindo:

- alta disponibilidade  
- escalabilidade  
- tolerância a falhas  

---

### 🔒 Internal Application Load Balancer

- Funciona **apenas dentro da rede privada (VPC)**  
- Não expõe VMs para a internet  
- Ideal para comunicação entre serviços internos  

👉 Uso comum:
- backend services
- microservices
- APIs internas

---

### 🧩 Componentes principais

| Componente        | Função |
|------------------|--------|
| Forwarding Rule  | IP interno que recebe requisições |
| Backend Service  | Define como o tráfego é distribuído |
| Health Check     | Verifica se as VMs estão saudáveis |

---

### 🖥️ Managed Instance Group (MIG)

- Grupo de VMs idênticas  
- Auto healing (recria instâncias se falharem)  
- Base para escalabilidade  

---

### 🌐 VPC (Virtual Private Cloud)

- Rede privada dentro do Google Cloud  
- Onde todos os recursos se comunicam  

---

## 🏗️ 2. Arquitetura do Lab

```bash
Usuário (Internet)
        ↓
Frontend (VM pública)
        ↓
Internal Load Balancer (IP privado)
        ↓
Backend MIG (3 VMs)
        ↓
Serviço interno (cálculo de números primos)
⚙️ 3. Passo a Passo Explicado
🔹 Etapa 1 – Ambiente virtual (Python)
sudo apt-get install -y virtualenv
python3 -m venv venv
source venv/bin/activate

📌 Isola dependências do projeto

🔹 Etapa 2 – Backend (MIG)
📄 Criar script backend

Arquivo backend.sh:

cria servidor HTTP em Python
retorna se número é primo
🧱 Criar template
gcloud compute instance-templates create primecalc \
--metadata-from-file startup-script=backend.sh \
--no-address --tags backend --machine-type=e2-medium

📌 Importante:

--no-address → VM sem IP público (segurança)
🔥 Liberar firewall
gcloud compute firewall-rules create http \
--network default \
--allow=tcp:80 \
--source-ranges IP \
--target-tags backend
⚙️ Criar instance group
gcloud compute instance-groups managed create backend \
--size 3 \
--template primecalc \
--zone ZONE

📌 Cria 3 VMs automaticamente

🔹 Etapa 3 – Load Balancer Interno
❤️ Health Check
gcloud compute health-checks create http ilb-health \
--request-path /2

📌 Testa se backend responde corretamente

🔗 Backend Service
gcloud compute backend-services create prime-service \
--load-balancing-scheme internal \
--region=REGION \
--protocol tcp \
--health-checks ilb-health
➕ Conectar MIG ao backend
gcloud compute backend-services add-backend prime-service \
--instance-group backend \
--instance-group-zone=ZONE \
--region=REGION
🎯 Forwarding Rule (IP interno)
gcloud compute forwarding-rules create prime-lb \
--load-balancing-scheme internal \
--ports 80 \
--network default \
--region=REGION \
--address IP \
--backend-service prime-service

📌 Esse é o IP interno do serviço

🔹 Etapa 4 – Teste interno

Criar VM de teste:

gcloud compute instances create testinstance \
--machine-type=e2-standard-2 --zone ZONE

Conectar via SSH:

gcloud compute ssh testinstance --zone ZONE

Testar:

curl IP/2
curl IP/4
curl IP/5

📌 Resultado esperado:

2 → True
4 → False
5 → True
🔹 Etapa 5 – Frontend público
📄 Criar script frontend

Arquivo frontend.sh:

chama o load balancer interno
gera página HTML com números primos
🖥️ Criar VM frontend
gcloud compute instances create frontend \
--zone=ZONE \
--metadata-from-file startup-script=frontend.sh \
--tags frontend \
--machine-type=e2-standard-2
🌍 Liberar acesso público
gcloud compute firewall-rules create http2 \
--network default \
--allow=tcp:80 \
--source-ranges 0.0.0.0/0 \
--target-tags frontend

📌 Acesse no navegador:

http://EXTERNAL_IP
⚠️ 4. Erros Comuns + Soluções
❌ 1. Backend não responde

✔️ Verifique:

kubectl / curl / logs

✔️ Possíveis causas:

script não rodou
porta errada
❌ 2. Health check falhando

✔️ Solução:

--request-path /2

👉 precisa retornar 200 OK

❌ 3. Load balancer não responde

✔️ Verifique:

backend service conectado
forwarding rule correta
IP interno correto
❌ 4. Firewall bloqueando

✔️ Backend:

--allow=tcp:80

✔️ Frontend:

--source-ranges 0.0.0.0/0
❌ 5. curl não funciona

✔️ Certifique-se:

está dentro da VM (não Cloud Shell)
usando IP interno
❌ 6. Página frontend não abre

✔️ Verifique:

IP externo correto
firewall liberado
script rodando
❌ 7. Timeout ao usar números grandes

📌 Normal (limitação do script)

Internet → Frontend VM → Internal LB → Backend MIG → Serviço
