🧠 Task 6 — Query Plans
🔍 O que é um Query Plan?

Plano de execução da query:

Como os dados serão acessados
Ordem das operações
Distribuição entre servidores
⚙️ Como funciona no Spanner
Query é compilada
Vai para um root server
Root:
Distribui subqueries
Coleta resultados
Retorna resposta

📌 Execução distribuída (árvore)

📊 Tipos de operações no plano
Table Scan → leitura da tabela
Index Scan → leitura via índice
Hash Aggregate → agregação (COUNT, SUM)
Distributed Union → execução distribuída
Serialize Result → ordena resultado final
📈 Exemplo: Query com agregação
SELECT ProductId, COUNT(*)
FROM Product
GROUP BY ProductId;
Fluxo:
Scan
Distributed execution
Aggregation parcial
Aggregation final
Resultado
🔗 Co-located Joins

Quando tabelas são interleaved:

👉 Dados relacionados ficam no mesmo nó

Vantagem:
Join mais rápido
Sem tráfego entre servidores
Exemplo:
SELECT c.CategoryName, pr.ProductName
FROM Category c, Product pr
WHERE c.PortfolioId = pr.PortfolioId
AND c.CategoryId = pr.CategoryId;

📌 Executado localmente (mais eficiente)
