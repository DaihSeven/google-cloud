# Laboratório: Deploy de Aplicação Web com Cloud Spanner GSP1051

Este laboratório demonstra como implantar uma aplicação web moderna conectada a um banco de dados utilizando Google Cloud.
<i> https://www.skills.google/catalog_lab/5729?locale=pt_BR</i>

### A aplicação utilizada é o OmegaTrade, composta por:

Frontend: Angular

Backend: Node.js + Express

Banco de dados: Cloud Spanner

Deploy: Cloud Run

Containers: Docker

## Arquitetura da aplicação:

Frontend (Angular)
      ↓
Cloud Run
      ↓
Backend (Node.js API)
      ↓
Cloud Spanner Database

## Objetivos do Laboratório

O objetivo deste laboratório é demonstrar como implantar uma aplicação web moderna na nuvem utilizando os serviços da Google Cloud, com foco na integração entre containers, serviços gerenciados e banco de dados distribuído.

Durante a execução do laboratório foi implantada a aplicação OmegaTrade, uma ferramenta de visualização de preços de ações que utiliza uma arquitetura baseada em microsserviços.

### Os principais objetivos foram:

Configurar o ambiente de desenvolvimento no Google Cloud.

Clonar e analisar o código da aplicação disponibilizada no repositório oficial de treinamento.

Implantar o serviço de backend desenvolvido em Node.js utilizando o framework Express.js.

Criar e configurar uma instância do banco de dados Cloud Spanner.

Importar dados de exemplo de negociação de ações para o banco de dados.

Criar imagens de container utilizando Docker.

Implantar os serviços de backend e frontend utilizando Cloud Run.

Configurar o frontend da aplicação desenvolvido em Angular.

Realizar operações na aplicação OmegaTrade conectada ao banco de dados na nuvem.

Ao final do laboratório, a aplicação fica totalmente executando em ambiente de nuvem, demonstrando a integração entre serviços gerenciados, containers e banco de dados distribuído.

## Arquitetura do Sistema

A aplicação OmegaTrade utiliza uma arquitetura baseada em separação de camadas, onde frontend, backend e banco de dados são executados de forma independente e escalável.

Diagrama da Arquitetura

                Usuário
                   │
                   │ HTTPS
                   ▼
        Frontend Angular (OmegaTrade)
              Cloud Run
                   │
                   │ REST API
                   ▼
        Backend Node.js + Express
              Cloud Run
                   │
                   │ Conexão segura
                   ▼
           Cloud Spanner Database
                   │
                   ▼
         Dados de ações (stocks)
### Explicação da Arquitetura

A arquitetura da aplicação é composta por três camadas principais: interface do usuário, camada de serviços e camada de dados.

#### 1. Camada de Interface (Frontend)

O frontend da aplicação foi desenvolvido utilizando o framework Angular.

Essa camada é responsável por:

exibir a interface da aplicação

permitir cadastro e login de usuários

mostrar gráficos de preços de ações

enviar requisições HTTP para o backend

O frontend foi empacotado em um container Docker e implantado no serviço Cloud Run, que executa aplicações containerizadas de forma escalável e totalmente gerenciada.

#### 2. Camada de Aplicação (Backend)

O backend da aplicação foi desenvolvido utilizando Node.js com o framework Express.

Essa camada é responsável por:

processar requisições da aplicação

implementar a lógica de negócio

gerenciar autenticação de usuários

consultar e gravar dados no banco

O backend também foi empacotado em um container Docker e implantado no Cloud Run, permitindo que o serviço escale automaticamente conforme o número de requisições.

#### 3. Camada de Dados

A camada de dados utiliza o banco relacional distribuído Cloud Spanner.

O Cloud Spanner é um serviço gerenciado que oferece:

    escalabilidade horizontal

    alta disponibilidade

    consistência transacional

    replicação global de dados

No laboratório, o banco armazena informações relacionadas a:

      empresas

      simulações de investimento

      preços de ações

Esses dados são acessados pelo backend por meio do cliente oficial do Cloud Spanner para Node.js.

### Fluxo de Funcionamento da Aplicação

O fluxo de funcionamento da aplicação ocorre da seguinte forma:

O usuário acessa a aplicação web através do navegador.

O frontend Angular envia requisições HTTP para a API do backend.

O backend Node.js processa as requisições e executa consultas no banco.

O banco Cloud Spanner retorna os dados solicitados.

O backend envia a resposta ao frontend.

O frontend exibe os dados ao usuário através de gráficos e tabelas.
