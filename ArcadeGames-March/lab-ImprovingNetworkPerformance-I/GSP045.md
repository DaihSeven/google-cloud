# 🚀 Improving Network Performance I (Google Cloud)

---

## 🧠 1. Conceitos Fundamentais (ANTES do lab)

### 🌐 Throughput (Taxa de Transferência)

- Quantidade de dados transferidos por segundo  
- Medido em:
  - Mbps (Megabits por segundo)
  - Gbps (Gigabits por segundo)

---

### 🔧 iperf

Ferramenta open-source usada para:

- medir throughput de rede  
- testar desempenho entre máquinas  
- simular tráfego  

👉 Funciona com:
- servidor (`iperf -s`)
- cliente (`iperf -c`)

---

### ⚙️ Relação: CPU vs Rede

📌 No Google Cloud:

- Cada **vCPU ≈ 2 Gbps de throughput**
- Limite máximo ≈ 16 Gbps por VM

👉 Ou seja:
- mais CPU → mais banda de rede

---

### 🌐 IP Interno vs IP Externo

| Tipo        | Característica | Performance |
|------------|----------------|------------|
| Internal IP | Dentro da VPC | 🔥 Mais rápido |
| External IP | Sai para internet | 🐢 Mais lento |

⚙️ 3. Passo a Passo Explicado
🔹 Etapa 1 – Teste básico de conectividade
ping -c 5 <IP>

📌 Verifica:

latência
perda de pacotes
🔹 Etapa 2 – Teste com iperf (1 vCPU)
Servidor:
iperf -s
Cliente:
iperf -c <IP>

📌 Resultado típico:

~ 4 Gbps (máximo aproximado)

👉 Limitação:

apenas 1 vCPU
🔹 Etapa 3 – Teste com máquina menor (micro)

Resultado:

~ 2 Gbps

📌 Insight:

máquinas menores = menos throughput
🔹 Etapa 4 – Teste com 4 vCPU
iperf -s
iperf -c <IP>

Resultado:

~ 7–8 Gbps

👉 Melhor desempenho devido a mais CPUs

🔹 Etapa 5 – Teste com múltiplas threads
iperf -c <IP> -P4
iperf -c <IP> -P8

📌 Resultado:

maior utilização da rede
pode atingir limite da VM
🔹 Etapa 6 – Teste Internal vs External IP
❌ External IP
iperf -c <external-ip>

Resultado:

~ 1 Gbps
✅ Internal IP
iperf -c <internal-ip>

Resultado:

~ 2 Gbps ou mais

📌 Conclusão:

Internal IP é mais rápido (sem sair da rede)
⚠️ 4. Erros Comuns + Soluções
❌ 1. iperf não conecta

✔️ Verifique firewall:

tcp:5001
udp:5001
❌ 2. Baixo throughput

✔️ Possíveis causas:

poucos vCPUs
uso de IP externo
CPU saturada
❌ 3. Resultado inconsistente

✔️ Solução:

rodar várias vezes
usar threads:
iperf -c <IP> -P8
❌ 4. CPU alta

✔️ Verifique:

uso de CPU (top / htop)
gargalo não é rede, é CPU
❌ 5. Teste fora da VPC

✔️ Problema:

Cloud Shell ≠ mesma rede

✔️ Solução:

usar VM interna
🧩 5. Insights Importantes (ESSENCIAL PRA PROVA)
🔥 1. CPU impacta rede
1 vCPU ≈ 2 Gbps
mais CPU = mais throughput
🔥 2. Internal IP é mais rápido
não sai da rede
menor latência
maior throughput
🔥 3. Threads aumentam performance
melhor uso da banda
evita gargalo de single-thread
🔥 4. Nem sempre o problema é rede

Pode ser:

CPU
configuração
arquitetura
🧩 6. Resumo Final (cola pra prova)
iperf → mede performance
CPU limita throughput
Internal IP > External IP
Threads melhoram desempenho
